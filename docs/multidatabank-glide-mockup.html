How to Build It ¬∑ 1 Day

1

Restructure the Google Sheet (2 hours)

The existing sheet needs three clean tabs: Slots (one row per slot with date, time, capacity), Bookings (one row per booking, written by the app), and Stock (one row per item with an available yes/no column). This is the "database" ‚Äî Glide reads and writes directly to it.

Tabs: Slots ¬∑ Bookings ¬∑ Stock ¬∑ Partners
2

Connect to Glide (30 mins)

Go to glideapps.com, create a free account, click "New App from Google Sheets", and paste in the sheet link. Glide immediately reads all columns and builds a basic app automatically. No code at all.

3

Configure the Partner View (3 hours)

Build the slot listing screen (filter by date, show available count), the booking form (name + reference ‚Üí writes a row to Bookings), and the stock screen (read-only list from the Stock tab). Set a "Row Ownership" rule so each partner only sees their own bookings.

4

Configure the Admin View (2 hours)

A separate tab in the app (password-protected or email-restricted) shows today's full booking list, allows marking arrivals, and lets warehouse staff toggle stock availability live. This writes directly back to the Stock tab in the sheet.

5

Add Email Confirmations via Apps Script (1 hour)

A small Google Apps Script triggers when a new row appears in the Bookings tab and sends a confirmation email to the partner. This is the one piece of lightweight code ‚Äî about 15 lines of JavaScript, nothing complex.

function onFormSubmit(e) { MailApp.sendEmail(...) }
